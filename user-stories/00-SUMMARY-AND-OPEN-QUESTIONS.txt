4. OPEN QUESTIONS & CLARIFICATIONS NEEDED
================================================================================

The following ambiguities were identified during analysis. These must be
resolved with the BA/Stakeholder before sprint planning.

--- UI LAYOUT / PANEL BEHAVIOR ---

Q1.  [US-01] Is the Expression panel a modal popup, inline dropdown, or a
     side panel? What triggers dismissal (outside click, Escape key)?

Q2.  [US-01] Which tab (Parameter / Math Function / All) is selected by
     default when the Expression panel first opens?

Q3.  [US-05] In the "All" tab, are Parameters and Math Functions shown as
     separate labeled sections or a single flat merged list?

Q4.  [US-05] Does the "All" tab have a search box similar to the Parameter tab?


--- PARAMETER / FIELD DATA ---

Q5.  [US-02, US-21] Are the field labels FieldA–FieldF static/hardcoded in
     the UI, or must they be fetched dynamically from a backend API?

Q6.  [US-02] What is the display order of fields in the Parameter list?

Q7.  [US-03] Should "Number" appear in search results when the user types
     "number" in the Parameter search box?

Q8.  [US-23] What are the data types of FieldA–FieldF (numeric, date, string)?


--- EXPRESSION BUILDING ---

Q9.  [US-04] When a non-BRACKET math function (e.g., MIN, CEIL) is added to
     the expression, what exact format is inserted into Add Expression —
     e.g., "MIN(...)" with placeholder, or just "MIN" as a token?

Q10. [US-06] Can the same field be used more than once in a single expression?

Q11. [US-07] What are the exact math operator symbols available in the "+"
     operator popup? (e.g., +, -, *, /, %)

Q12. [US-07/08] Is the flow two separate steps (one popup for operator, then
     panel for operand), or a single combined popup containing all of this?

Q13. [US-10] Is there input validation on the number entry popup (e.g., only
     numeric values allowed, decimal precision limit, min/max range)?

Q14. [US-10] Can the user re-click a previously saved numeric token to edit
     it again?

Q15. [US-10] Does the number entry popup have a "Cancel" button in addition
     to "Save"?

Q16. [US-11] Can BRACKET be placed as the very first element of an expression
     (before any operand), or only after a left operand + operator?

Q17. [US-11] Does clicking BRACKET replace the entire existing expression, or
     insert the bracket at the current cursor/position within the expression?

Q18. [US-12] Is there a maximum depth limit for bracket nesting?

Q19. [US-12] Can individual tokens inside a bracket be deleted independently?

Q20. [US-13] Can there be multiple brackets at the same level in one expression
     (e.g., (A + B) * (C + D))?


--- COMPARATOR ---

Q21. [US-15] Should "Not equal to" be included as a comparator option?
     It is not mentioned in the document.

Q22. [US-15] After a comparator is selected, can the user click it to change
     it to a different comparator without rebuilding the condition?

Q23. [US-14] Is the "Add Comparator" button available before the left-hand
     expression is built, or is it only enabled after the left side has content?

Q24. [US-16] Is the right-hand expression built in the same Add Expression
     component or a visually separate component?


--- CONDITION MANAGEMENT ---

Q25. [US-17] Is there an explicit "Save Condition" or "Apply" button, or is
     the condition auto-completed when all three parts are filled?

Q26. [US-17] What happens to partial conditions (only left side filled) — are
     they preserved or discarded on panel close or navigation?

Q27. [US-18] Is there a confirmation dialog before deleting a condition,
     or is deletion instant?

Q28. [US-18] If a condition inside a subgroup is deleted and it is the last
     condition in the subgroup, does the subgroup container also collapse/remove?


--- AND/OR LOGIC & GROUPING ---

Q29. [US-19] Is the "(+ and/or)" button available inside a subgroup as well,
     or only at the top level?

Q30. [US-19] Can AND/OR connectors between different condition pairs be set
     independently (e.g., Condition1 AND Condition2 OR Condition3)?

Q31. [US-20] Is grouping triggered by clicking the AND/OR toggle (as implied
     by the document), or is there a dedicated "Group" button?

Q32. [US-20] Can an existing condition be moved into or out of a subgroup
     (drag-and-drop or any interaction)?

Q33. [US-20] Is subgroup nesting truly unlimited, or is there a depth cap?

Q34. [US-20] Does each subgroup have its own delete control (bin icon), or
     is it removed by deleting all its child conditions?


--- BACKEND & PERSISTENCE ---

Q35. [US-22] Is there a "Save Rule" button in the UI? The document does not
     mention one. Are rules auto-saved or require explicit save?

Q36. [US-22] Can a previously saved rule be loaded back into the builder for
     editing?

Q37. [US-22] What is the expected JSON schema/structure for persisting a rule?

Q38. [US-23] Is expression evaluation done in real-time (backend call while
     building) or on explicit user action?

Q39. [General] What does "RBP" stand for in "RBP Calculator"?

Q40. [General] What is the broader business use case for this rule builder
     (e.g., pricing engine, eligibility check, risk scoring)?

       tokens, and numeric tokens in the expression display.

IR-04  The ability to delete individual tokens from an expression
       (e.g., removing one field or operator mid-expression).
       [Needs Clarification — not mentioned in document]

IR-05  Responsive/overflow behavior when a built expression is very long.

IR-06  Loading state for the Parameter list if fields are fetched from an API.

IR-07  Error state handling if the backend API (field list / save / evaluate)
       fails or is unavailable.
