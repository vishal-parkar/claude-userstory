================================================================================
EPIC 2: Expression Building
================================================================================
Scope   : Frontend
Stories : US-06, US-07, US-08, US-09, US-10, US-11, US-12, US-13
Goal    : Allow the user to construct a complete expression by combining fields,
          math functions, operators, numbers, and brackets inside the
          "Add Expression" field.
================================================================================


--------------------------------------------------------------------------------
US-06 | Add a Field/Parameter to the Expression
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click a field from the Parameter list,
So that the selected field is added to the "Add Expression" area as the
first operand.

Acceptance Criteria:
- Clicking a field (FieldA–FieldF) from the Parameter panel adds the field name
  as a token/chip/label in the "Add Expression" field.
- A "+" (plus sign) button appears alongside or after the added field token
  in the Add Expression area.
- If the expression is empty, the clicked field becomes the first token.
- Only one field is added per click.
- The selected field token is visually distinct (e.g., pill/chip style).

Notes / Clarifications:
- [Needs Clarification] Can the same field be added more than once in a single
  expression?
- [Needs Clarification] Is there a visual difference between a field token and
  a math function token in the expression?


--------------------------------------------------------------------------------
US-07 | Add Operator via Plus Sign
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click the "+" (plus sign) in the Add Expression area,
So that I can select a math operator and then a second operand to extend
the expression.

Acceptance Criteria:
- Clicking the "+" (plus sign) opens a popup/dropdown.
- The popup displays math operator options (e.g., +, -, *, /).
- After selecting a math operator, the operator is added to the Add Expression
  as a token.
- After adding the operator, the same Parameter/Math Function/All panel
  reopens automatically so the user can select the right-side operand.
- The selected operand is added to the expression after the operator.
- The "+" sign reappears after each new operand is added.

Notes / Clarifications:
- [Needs Clarification] What are the exact math operator options available in
  the operator popup (e.g., +, -, *, /, %)?
- [Needs Clarification] Is selecting an operator mandatory before selecting
  the next operand, or can the user cancel mid-flow?


--------------------------------------------------------------------------------
US-08 | Operator Popup Contains Parameter / Math Function / All Options
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want the operator-selection popup (triggered by the "+" sign) to also allow
me to choose between Parameter, Math Function, and All,
So that after selecting an operator I can pick the next operand from the
same familiar panel structure.

Acceptance Criteria:
- After a math operator is selected from the "+" popup, the panel that
  re-opens mirrors the same three options: Parameter, All, Math Function.
- In this re-opened panel, the "Parameter" tab includes:
    - FieldA, FieldB, FieldC, FieldD, FieldE, FieldF
    - Number (see US-09)
- The "Math Function" tab includes: Min, MAX, FLOOR, ABS, BRACKET, CEIL.
- The "All" tab shows the combined list.
- Selecting any item from this panel adds it to the expression as the
  next operand.
- This cycle (+ → operator → operand panel) repeats indefinitely to allow
  building long expressions.

Notes / Clarifications:
- [Needs Clarification] The document mentions clicking "+" opens a popup with
  math operators, and THEN the parameter/math function panel appears. Confirm
  whether these are two separate steps (one popup for operator, then panel for
  operand) or a single combined popup.


--------------------------------------------------------------------------------
US-09 | Number Option in Parameter Panel
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to select "Number" from the Parameter panel,
So that I can add a numeric constant value to the expression.

Acceptance Criteria:
- "Number" is listed as an option within the Parameter panel (alongside
  FieldA–FieldF).
- "Number" also appears in the "All" tab combined list.
- Clicking "Number" adds the token "0.0" to the Add Expression field.
- The "0.0" token is displayed as a clickable/editable element in the
  expression.
- The "+" plus sign appears after "0.0" is added, consistent with other
  operand additions.

Notes / Clarifications:
- [Needs Clarification] Is "Number" also filterable via the search box in
  the Parameter panel?


--------------------------------------------------------------------------------
US-10 | Edit Numeric Value in Expression
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click on the "0.0" token in the Add Expression field,
So that I can replace it with a specific numeric value of my choosing.

Acceptance Criteria:
- Clicking on the "0.0" token opens a popup/dialog.
- The popup contains a numeric input field.
- The user enters a number (integer or decimal) in the input field.
- The popup has a "Save" button.
- Clicking "Save" closes the popup and replaces "0.0" in the expression with
  the entered value (e.g., "10.0").
- The entered value is reflected immediately in the Add Expression field.
- If the user closes the popup without saving, "0.0" remains unchanged.

Notes / Clarifications:
- [Needs Clarification] Is there input validation (e.g., only numeric values
  allowed, min/max limits)?
- [Needs Clarification] Can the user edit a previously saved number again by
  clicking it?
- [Needs Clarification] Is there a "Cancel" button in the number input popup,
  or only "Save"?


--------------------------------------------------------------------------------
US-11 | Bracket Math Function — Wrap Expression in Brackets
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to select "BRACKET" from the Math Function panel,
So that I can create a grouped sub-expression enclosed in parentheses/brackets
within my expression.

Acceptance Criteria:
- Clicking "BRACKET" in the Math Function panel replaces/updates the
  Add Expression display to show a bracketed container (e.g., "( )").
- Inside the bracket container, a "+" (plus sign) is available to add operands
  and operators within the bracket (see US-12).
- Outside the bracket container, a "+" (plus sign) is available to add
  operands and operators outside the bracket (see US-13).
- The bracket visually groups the inner expression separately from the
  outer expression.

Notes / Clarifications:
- [Needs Clarification] Can BRACKET be selected as the very first element of
  an expression (before any operand), or only after an operator?
- [Needs Clarification] What does "Add Expression will get replaced" mean — is
  the entire expression replaced, or is BRACKET inserted at the current
  cursor/position?


--------------------------------------------------------------------------------
US-12 | Build Expression Inside Bracket (Nested Expression)
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to use the "+" inside the bracket to add fields and operators within
the bracketed sub-expression,
So that I can build a grouped nested expression inside the bracket.

Acceptance Criteria:
- Clicking the "+" inside the bracket opens the same Parameter/Math
  Function/All panel.
- The item selected (field or function) is added inside the bracket as the
  left operand.
- Clicking "+" again inside the bracket opens the operator popup, then
  re-opens the operand panel for the right operand.
- This inside-bracket cycle (add operand → + → operator → operand) can
  repeat indefinitely to build a full sub-expression inside the bracket.
- Items added inside the bracket are visually scoped within the bracket
  boundaries.
- BRACKET can be nested (clicking BRACKET inside the bracket creates
  another level of brackets).

Notes / Clarifications:
- [Needs Clarification] Is there a limit on the depth of bracket nesting?
- [Needs Clarification] Can the user delete individual tokens inside the
  bracket, or only the entire bracket?


--------------------------------------------------------------------------------
US-13 | Build Expression Outside Bracket
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to use the "+" outside the bracket to add operators and operands
after the bracketed expression,
So that I can continue building the expression beyond the bracket scope.

Acceptance Criteria:
- A "+" (plus sign) is available outside/after the closing bracket.
- Clicking this outer "+" opens the operator popup first.
- The user selects a math operator.
- After operator selection, the Parameter/Math Function/All panel opens for
  selecting the right-side operand.
- The selected operand is added to the expression after the bracket and operator.
- The flow follows the same cycle as US-07 and US-08.
- The outer "+" continues to appear after each new addition for further
  extension of the expression.

Notes / Clarifications:
- [Needs Clarification] Can there be multiple brackets at the same level in
  one expression (e.g., (A + B) * (C + D))?
