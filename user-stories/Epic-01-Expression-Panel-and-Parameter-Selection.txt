================================================================================
EPIC 1: Expression Panel & Parameter Selection
================================================================================
Scope   : Frontend
Stories : US-01, US-02, US-03, US-04, US-05
Goal    : Allow the user to open the expression builder panel and browse/filter
          available parameters (fields) and math functions before adding them
          to an expression.
================================================================================


--------------------------------------------------------------------------------
US-01 | Open Expression Panel
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click on the "Expression" label to open the expression builder panel,
So that I can begin constructing a mathematical or logical expression.

Acceptance Criteria:
- Clicking the "Expression" label/button opens a panel or dropdown overlay.
- The panel displays three selectable options/tabs: "Parameter", "Math Function",
  and "All".
- The panel opens in a consistent position relative to the "Expression" trigger.
- The panel is dismissible (e.g., clicking outside closes it).
- No "1Min" dropdown is present anywhere in the expression panel or the
  surrounding screen.

Notes / Clarifications:
- [Needs Clarification] Is the panel a modal popup, an inline dropdown, or a
  side panel?
- [Needs Clarification] Should the panel default to a specific tab
  (Parameter / All / Math Function) on first open?


--------------------------------------------------------------------------------
US-02 | Display Parameter List with Correct Label
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to see all available fields listed under the "Parameter" option,
So that I can identify and select the correct field for my expression.

Acceptance Criteria:
- Clicking/selecting "Parameter" inside the expression panel displays the
  following fields as selectable items:
    - FieldA
    - FieldB
    - FieldC
    - FieldD
    - FieldE
    - FieldF
- The section/tab label is displayed as "Parameter" (NOT "Indicators").
- Each field is displayed as a clickable item.
- The "Number" option is also present in the Parameter list (see US-09 for
  Number behavior).
- "Math Function" label is NOT renamed — it retains its original label.
- The "All" option label is NOT renamed — it retains its original label.

Notes / Clarifications:
- [Needs Clarification] Are the field labels (FieldA–FieldF) static/hardcoded
  in the UI, or are they fetched dynamically from a backend API?
- [Needs Clarification] What is the display order of the fields?


--------------------------------------------------------------------------------
US-03 | Filter Parameters by Search
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to type a field name in a search box within the Parameter panel,
So that I can quickly find a specific parameter without scrolling through
the full list.

Acceptance Criteria:
- A search/filter text input is displayed inside the Parameter panel.
- As the user types, the field list filters in real-time to show only fields
  whose names contain the typed text (case-insensitive match recommended).
- If no field matches the search text, the list shows an empty state
  (no results).
- Clearing the search box restores the full field list.
- The "Number" option behavior under search:
  [Needs Clarification] Should "Number" also appear in search results when
  searching by the word "number"?

Notes / Clarifications:
- [Needs Clarification] Should the search also filter across the Math Function
  options when the "All" tab is active?


--------------------------------------------------------------------------------
US-04 | Display Math Function Options
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click "Math Function" to view available mathematical functions,
So that I can select a function to apply in my expression.

Acceptance Criteria:
- Clicking "Math Function" within the expression panel displays exactly the
  following options as selectable items:
    - Min
    - MAX
    - FLOOR
    - ABS
    - BRACKET
    - CEIL
- The section/tab label remains "Math Function" (unchanged).
- Each function name is displayed exactly as listed above (case-sensitive).
- Each function is a clickable item.
- Selecting BRACKET has a distinct behavior (see US-11 – Bracket Expression).
- Selecting any other function (Min, MAX, FLOOR, ABS, CEIL) adds the function
  to the Add Expression field.

Notes / Clarifications:
- [Needs Clarification] When a non-BRACKET math function (e.g., MIN, CEIL) is
  added to the expression, what is the exact format/syntax added to the
  Add Expression field (e.g., MIN(...) or just MIN)?


--------------------------------------------------------------------------------
US-05 | Display All Options Combined (All Tab)
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click "All" to see all Parameters and Math Functions together in
one list,
So that I can find any option quickly without switching between tabs.

Acceptance Criteria:
- Clicking "All" displays a combined list of all Parameter fields
  (FieldA–FieldF + Number) and all Math Functions (Min, MAX, FLOOR, ABS,
  BRACKET, CEIL) in a single scrollable list.
- The label "All" is displayed as-is (unchanged).
- Clicking any item in the "All" list triggers the same behavior as clicking
  that item in its respective tab (Parameter or Math Function).
- If the user was previously on "Parameter" or "Math Function" and clicks
  "All", the full combined list is shown.
- Clicking "Parameter" or "Math Function" after "All" switches back to the
  respective filtered view.

Notes / Clarifications:
- [Needs Clarification] In the "All" view, are Parameters and Math Functions
  displayed as separate sections/groups or as a flat merged list?
- [Needs Clarification] Is there a search box available in the "All" tab as
  well, similar to the Parameter tab?
