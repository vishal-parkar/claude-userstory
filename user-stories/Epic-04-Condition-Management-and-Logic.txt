================================================================================
EPIC 4: Condition Management & AND/OR Logic
================================================================================
Scope   : Frontend
Stories : US-17, US-18, US-19, US-20
Goal    : Allow the user to complete a condition, delete it, chain multiple
          conditions with AND/OR logic, and group conditions into subgroups
          with unlimited nesting depth.
================================================================================


--------------------------------------------------------------------------------
US-17 | Complete Condition Creation
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to see a complete condition row formed from a left expression,
a comparator, and a right expression,
So that I have a full readable condition displayed in the rule builder.

Acceptance Criteria:
- A condition row consists of three parts in sequence:
    1. Left-hand Add Expression (built via US-06 through US-13)
    2. Add Comparator (selected via US-14 through US-15)
    3. Right-hand Add Expression (built via US-16)
- Once all three parts are populated, the condition is visually displayed
  as a single complete row.
- The condition row shows all tokens (field names, operators, comparator,
  numeric values) in a readable inline format.
- A delete (bin/trash) icon is displayed alongside the condition row
  (see US-18).

Notes / Clarifications:
- [Needs Clarification] Is there a "Save" or "Apply" action for a condition,
  or is it considered complete as soon as all three parts are filled?
- [Needs Clarification] Should partial conditions (e.g., only left expression
  filled) be persisted/visible or cleared on navigation?


--------------------------------------------------------------------------------
US-18 | Delete a Condition
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click the delete (bin) icon on a condition row,
So that I can remove it and return to the initial empty rule builder state.

Acceptance Criteria:
- Each condition row displays a delete/bin icon.
- Clicking the bin icon removes the condition from the rule builder.
- After deletion, the screen resets to the initial empty state (as it appeared
  before any condition was added).
- If there was only one condition, the full rule builder resets to empty.
- If there are multiple conditions, only the targeted condition is deleted;
  the remaining conditions stay intact.

Notes / Clarifications:
- [Needs Clarification] Is there a confirmation dialog before deleting a
  condition, or is it instant?
- [Needs Clarification] If a condition inside a subgroup is deleted, does the
  subgroup container remain or also collapse?


--------------------------------------------------------------------------------
US-19 | Add New Condition with AND/OR Logic
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to click the "(+ and/or)" button to add a new condition line,
So that I can chain multiple conditions together using AND or OR logic.

Acceptance Criteria:
- A "(+ and/or)" button/label is available in the rule builder.
- Clicking "(+ and/or)" adds a new empty condition row below the existing one.
- The new condition row has its own:
    - Left-hand Add Expression
    - Add Comparator
    - Right-hand Add Expression
    - Delete (bin) icon
- A logic connector label ("and" or "or") is displayed between consecutive
  condition rows.
- The default connector between conditions is "and".
- Clicking the "and" connector label toggles it to "or".
- Clicking the "or" connector label toggles it back to "and".
- The "(+ and/or)" button remains available to add further condition rows.

Notes / Clarifications:
- [Needs Clarification] Is the "(+ and/or)" button displayed inside or outside
  a subgroup?
- [Needs Clarification] Can the logic connector (and/or) between two specific
  conditions be set independently from other connectors in the same rule?


--------------------------------------------------------------------------------
US-20 | Group Conditions into Subgroups (Unlimited Nesting)
--------------------------------------------------------------------------------
Type : Frontend

As a rule builder user,
I want to group conditions into subgroups using a "Group" action,
So that I can create complex nested logic (e.g., (A AND B) OR (C AND D))
at any level of nesting.

Acceptance Criteria:
- A "Group" control/button is available to wrap a set of conditions into a
  subgroup container.
- Clicking the "and" toggle button when in OR context (or per document
  description) creates a subgroup condition with its own boundary/container.
- The subgroup visually groups its child conditions with a distinct border or
  indentation.
- Inside a subgroup, all the same actions are available:
    - Add condition rows (via + and/or)
    - Toggle AND/OR connectors between conditions
    - Delete individual conditions
    - Create further nested subgroups (unlimited depth)
- Subgroups can be nested to any level of depth.
- Each subgroup has its own AND/OR logic connector to the parent level.
- Deleting a subgroup's last condition collapses/removes the subgroup
  container.

Notes / Clarifications:
- [Needs Clarification] Is there a dedicated "Group" button, or is grouping
  triggered by the AND/OR toggle mechanism as described in the document
  ("To add 'or' condition use has to click on 'and' button once again. Group.
  This will create subgroup condition")?
- [Needs Clarification] Can an existing condition row be moved into or out of
  a subgroup via drag-and-drop or any other interaction?
- [Needs Clarification] Is there a maximum depth limit for subgroup nesting,
  or is it truly unlimited?
- [Needs Clarification] Does the subgroup have its own delete control, or do
  users delete it by deleting all conditions inside it?
